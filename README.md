CS 300 Portfolio Reflection
What was the problem you were solving in the projects for this course?

Throughout this course, I worked on solving problems related to organizing, storing, and retrieving data efficiently using various data structures. In Project One, I analyzed the performance and memory usage of vectors, hash tables, and binary search trees to determine which data structure would be most effective for managing course information. In Project Two, I developed a program that could load course data from a file, store it in an appropriate data structure, and display the courses in alphanumeric order while allowing users to search for specific course information.

How did you approach the problem? Consider why data structures are important to understand.

I approached these problems by first analyzing the requirements of the application and then selecting data structures that would provide efficient performance for searching, sorting, and storing information. Understanding data structures is important because they directly impact the efficiency, scalability, and maintainability of software applications. By comparing vectors, hash tables, and binary search trees, I gained a deeper understanding of how choosing the appropriate data structure can significantly improve program performance.

How did you overcome any roadblocks you encountered while going through the activities or project?

Throughout the course, I encountered several challenges related to debugging code, configuring my development environment, and implementing the required algorithms correctly. I overcame these obstacles by breaking problems into smaller steps, carefully reviewing error messages, testing my code frequently, and researching concepts when necessary. This process strengthened my problem-solving skills and taught me the importance of patience and persistence when developing software.

How has your work on this project expanded your approach to designing software and developing programs?

Working on these projects expanded my understanding of software design by emphasizing the importance of planning before implementation. I learned to evaluate different approaches, consider performance trade-offs, and develop programs with scalability and efficiency in mind. Additionally, I gained experience designing modular programs that separate functionality into manageable components, making the software easier to understand and maintain.

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

My experience in this course has changed the way I write programs by encouraging me to focus on readability, maintainability, and adaptability from the beginning of the development process. I now place greater emphasis on using descriptive variable names, organizing code into functions and classes, adding comments where appropriate, and designing solutions that can be modified or expanded in the future. These practices help create software that is easier to debug, maintain, and enhance over time.
